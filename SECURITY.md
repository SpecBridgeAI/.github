# Security policy

## Reporting a vulnerability

Please do not disclose security vulnerabilities in public issues.

Use the **Security** tab of the affected repository to submit a private vulnerability report when private vulnerability reporting is available. If the affected repository does not provide that option, use the controlled contact route published at [specbridge.co](https://specbridge.co).

Do not include customer data, credentials, access tokens, confidential documents or production secrets in an issue, pull request or public discussion.

## Scope

Security reports should identify the affected repository or service, the observed behavior, reproduction conditions using non-sensitive data, and the potential impact.

SpecBridge AI does not authorize testing that disrupts availability, accesses other users' data, bypasses authentication controls or targets third-party infrastructure.
